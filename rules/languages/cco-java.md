# Java Rules
*Stack-specific rules for Java projects*

**Trigger:** {java_manifest}, {java_ext}

## Null-Safety

Use Optional<T> for nullable returns

## Resource-Try

try-with-resources for AutoCloseable

## Immutable-Prefer

Prefer immutable objects, final fields

## Stream-API

Use Stream API for collection transformations

## Records

Use records for immutable data carriers

## Pattern-Switch

Use pattern matching in switch

## Virtual-Threads

Use virtual threads for high-concurrency I/O

## Sealed-Classes

Use sealed classes for controlled inheritance
