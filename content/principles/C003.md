# C003: Follow Existing Patterns

## Principle
Follow existing code patterns, naming conventions, and architectural decisions.

## Why
- Maintains consistency across codebase
- Reduces cognitive load for developers
- Makes code predictable and navigable
- Prevents conflicting styles

## How
- Before writing code, examine similar existing code
- Match naming conventions (camelCase, snake_case, etc.)
- Use same error handling patterns
- Follow established file organization
- Replicate testing patterns

## Examples

**Bad (inconsistent):**
```python
# Existing code uses snake_case
def process_data(data):
    pass

# New code introduces camelCase
def validateInput(input):  # ❌ Inconsistent
    pass
```

**Good (consistent):**
```python
# Existing code uses snake_case
def process_data(data):
    pass

# New code follows pattern
def validate_input(input):  # ✅ Consistent
    pass
```
