**Severity**: Medium

Explicit dependencies via DI, no globals, easier testing.

**Why**: Maintains code quality through consistent PR templates and review checklists

**Enforcement**: Skills required - verification_protocol, test_first, root_cause_analysis

**Project Types**: all
**Languages**: all

**Rules**:
- **No Global State**: No global variables

