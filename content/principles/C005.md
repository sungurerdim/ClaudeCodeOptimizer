# C005: Production-Grade Code Only

## Principle
Write production-ready code from the start. No TODO markers, debug prints, or placeholder code.

## Why
- Prevents technical debt accumulation
- Ensures code is always deployable
- Maintains high quality standards
- Avoids "temporary" code becoming permanent

## How
- Complete implementations fully
- Remove all debug/print statements
- No TODO/FIXME/HACK comments
- Proper error handling from start
- All code paths implemented

## Examples

**Bad:**
```python
def process_payment(amount):
    # TODO: Add error handling
    print(f"Processing {amount}")  # Debug
    result = payment_api.charge(amount)
    # FIXME: Handle edge cases
    return result
```

**Good:**
```python
def process_payment(amount):
    if amount <= 0:
        raise ValueError("Amount must be positive")

    try:
        result = payment_api.charge(amount)
        return result
    except PaymentError as e:
        logger.error(f"Payment failed: {e}")
        raise
```

## Exceptions
- Explicitly requested by user
- Part of documented development process
