# C001: No Unnecessary File Creation

## Principle
Never create files unless absolutely necessary for achieving your goal.

## Why
- Reduces codebase bloat
- Prevents maintenance burden
- Keeps project structure clean
- Forces consideration of existing solutions

## How
- Before creating a file, ask: "Can I edit an existing file instead?"
- Check if similar functionality exists elsewhere
- Prefer extending existing files over creating new ones
- Only create when genuinely required for the task

## Examples

**Bad:**
```python
# Creating new utils.py when helpers.py already exists
# utils.py
def format_date(date):
    return date.strftime("%Y-%m-%d")
```

**Good:**
```python
# Adding to existing helpers.py
def format_date(date):
    return date.strftime("%Y-%m-%d")
```

## Exceptions
- New feature genuinely requires new module
- Existing file would become too large (>500 lines)
- Clear separation of concerns demands it
